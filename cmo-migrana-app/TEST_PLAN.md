# Plan de pruebas

La suite de pruebas (`node --test`) cubre máximos, umbrales, prioridad automática, menores de edad, datos ausentes, override, intervenciones acumulativas, exclusión de conceptos prohibidos, importación/exportación e informe imprimible. Quedan pendientes pruebas E2E de navegador real y auditoría documental con el PDF fuente.

## Preestratificación automática desde texto (analizador local)

`test/local-extractor.test.mjs` (49 pruebas) cubre, entre otras: negaciones simples ("niega ansiedad"); negaciones con alcance limitado (una negación no contamina la cláusula siguiente; "sin embargo" no se interpreta como negación); antecedentes familiares (nunca se asignan al paciente); datos no mencionados (nunca se proponen como valor ni como cero); expresiones dudosas ("no puede descartarse embarazo", "posible efecto adverso"); información contradictoria (naïve + tratado simultáneos, HIT-6 alto + MIDAS bajo); cifras y rangos (edad, IMC); unidades (HIT-6, MIDAS); criterios que requieren duración temporal (migraña crónica ≥15 días/mes durante ≥3 meses, incluida la variante con el número escrito en letra "durante cuatro meses"; uso excesivo de medicación sintomática con los dos ejemplos exactos del encargo: "12 días este mes" queda dudoso, "12-14 días al mes durante cuatro meses" confirma el criterio); falta de adherencia ("olvida alguna dosis" dudoso, "buena adherencia" sin método dudoso, con método confirmado, "abandono del tratamiento" confirmado); embarazo/lactancia/deseo gestacional y sus negaciones; edad inferior a 18 años; pureza del analizador (no depende de estado externo, no modifica el texto de entrada); y dos pruebas de auditoría automática del propio código fuente que comprueban que `local-extractor.mjs` no contiene ninguna referencia a `fetch`/`XMLHttpRequest`/`axios` ni a dominios o credenciales de proveedores de IA (Azure/OpenAI/Anthropic/`apiKey`).

Además se ha verificado manualmente con un navegador real (Playwright/Chromium): pegar un evolutivo sintético → "Analizar texto y proponer valores" → tabla de revisión con variable, propuesta, certeza, fragmento, regla y puntuación provisional → confirmar/corregir/rechazar fila a fila → los campos confirmados quedan marcados «Preestratificación — revisar» en el cuestionario de abajo → editar manualmente un campo retira la marca → "Borrar todos los datos de este análisis" limpia todo → cuestionario manual sin usar el analizador sigue calculando la prioridad exactamente igual que antes → DevTools > Network no muestra ninguna petición al analizar (0 solicitudes) → DevTools > Application no muestra ningún dato del texto pegado ni de las propuestas en `localStorage`/`sessionStorage`/`IndexedDB`/cookies tras la sesión → usable en viewport 375×667 (la tabla de revisión, al ser ancha, se desplaza horizontalmente dentro de su propio contenedor sin desbordar la página).
